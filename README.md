# QuizNest

QuizNest 是一款基于本地优先架构（Local-First）的 AI 驱动学习与刷题平台。项目聚焦于高强度的知识提取、自动化测验与错题追踪，旨在构建闭环的个人学习工作流。

## 技术架构与数据存储

- **核心架构**：基于 Tauri 2 的跨端（macOS/Windows/Android）桌面与移动应用。
- **视图层**：原生 JavaScript 与 Vanilla CSS 构建，零现代前端框架依赖，以保证极致的渲染性能与轻量的内存占用。
- **AI 引擎**：全面兼容原生 OpenAI Chat Completions API（强制依赖 `response_format=json_object`），兼顾大语言模型的文本生成与视觉多模态（Vision）图文识别能力。
- **数据管理**：所有学习数据（包括配置、笔记、题库、错题本与聊天长期记忆）均强制持久化于本机的 IndexedDB。除向目标大模型服务端点发起推理请求外，系统无任何第三方云端数据同步，确保数据隔离与隐私安全。

## 核心学习工作流

QuizNest 的业务逻辑建立在以下严密的学习数据流转之上：

1. **知识提纯**：支持导入 `.md`、`.pdf` 以及 `.pptx`。本地通过抽取解析后，利用模型提炼知识骨架。
2. **自动化出题**：调用大模型将非结构化的知识转化为标准化题组（客观选择题 + 主观大题）。
3. **闭环测验**：提供强沉浸式的练习工作台。客观题即时保存并锁定，主观题支持图文混合输入并交由多模态模型批改评分。
4. **错题诊断**：自动追踪错因与薄弱点。错题本提供分层级的详细诊断信息（摘要优先，解析按需展开）。
5. **上下文解惑**：基于“错题-原题-答题记录”的完整数据结构，自动装载上下文发起对话，支持对单点知识的连续追问。

## 编译与开发指南

### 前端本地调试

```bash
# 启动本地 Node.js 代理与静态文件服务
npm start
```
*服务启动后，本地调试地址默认为 `http://localhost:5173`。*

### 自动化构建脚本

项目提供以下标准自动化脚本：

- `npm run sync-dist`：执行静态资源同步，将前端源码最新变更拷贝至 `dist/`，供 Tauri 后端调用。
- `npm run build:dmg`：执行完整的 macOS 本地打包流程（执行代码同步 -> Rust 编译 -> App 签名修正 -> DMG 镜像生成），最终产物输出至项目根目录。
- `npm run clean`：深度清理编译缓存（包含 `src-tauri/target` 与相关的临时构建文件）。

## 验收建议与故障排查

- **接口跨域/代理失败**：
  若设置页提示“本地代理无法访问外网”，排查重点为运行 `npm start` 的 Node 进程是否受到操作系统沙盒、防火墙或网络代理策略的拦截。
- **JSON 序列化失败**：
  若提示不支持 `response_format=json_object`，说明当前配置的模型端点未严格实现该规范。核心业务流（如生成题组）强制依赖该特性，请更换更可靠的基座模型。
- **请求超时（Timeout）**：
  生成大规模题组消耗时间较长。若频发网络超时，请在系统设置中将“请求超时”上限调整至 `180000` 到 `300000` 毫秒。
- **多模态特性约束**：
  “PPTX 导入出题”与“主观题上传图片”功能，强制要求配置中的目标模型原生具备视觉解析（Vision）能力并显式开启。
